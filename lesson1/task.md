## Задание

Придумать схему кодирования, максимально экономичную по памяти.

---

Символы для кодирования:
- кириллица нижний регистр = 33
- кириллица верхний регистр = 33
- цифры = 10
- знаки препинания = 15 (,.;:"'()[]-—!?…)
- управляющие символы (пробел, таб, перевод строки) = 3

Итого: 33 + 33 + 10 + 15 + 3 = 94

№1. Упрощенно можно закодировать этот набор символов в 7-битных последовательностях (2^7 = 128 > 94).

| Символ | Код     |
|--------|---------|
| а      | 0000001 |
| б      | 0000010 |
| ...    |         |
| \n     | 1011110 |

№2. Попробуем оптимизировать кодирование добавлением символа "перевод в верхний регистр". Тогда у нас останется 62 символа для кодирования.
Эти 62 символа можно закодировать 6 битами (2^6 = 64). Так получаем схему

| №   | Символ | Код    |
|-----|--------|--------|
| 1   | а      | 000000 |
| 2   | б      | 000001 |
| ... |        |        |
| 62  | \u     | 111101 |

№3. Можем пойти дальше и поиграться с изменением длины кода для некоторых символов.

Однако нужно выбрать, какие символы будут иметь короткий код. Для этого можно обратиться к статистике частоты использования русских букв.
Для наиболее часто используемых символов (о, е, а) берем короткие коды, далее пытаемся подобрать каждому символу код, который не будет являться началом другого кода в таблице.
Перевод в верхний регистр осуществляется добавлением перед кодом символа кода верхнего регистра (1111111).
